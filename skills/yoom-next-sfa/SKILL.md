# Next Sfa Yoom 연계 스킬

Next Sfa와 OpenClaw 연결을 위한 스킬입니다.

## 서비스 정보
- **서비스명**: Next Sfa
- **카테고리**: セール스 (Sales)
- **Yoom URL**: https://lp.yoom.fun/apps/next-sfa

## 연계 정보
- **연계 방식**: API
- **인증 방식**: OAuth
- **API 액션 수**: 28個
- **트리거 수**: 7個
- **템플릿 수**: 6個

## 구현된 API 액션
1. 매출 정보 삭제 (売上情報の削除)
2. 매출 정보 가져오기 (売上情報を取得)
3. 기업 담당자 정보 검색 (企業担当者情報を検索)
4. 사안 업데이트 (案件を更新)
5. 기업 담당자 정보 가져오기 (企業担当者情報を取得)
6. 기업 등록 (企業を登録)
7. 대응 이력 정보 검색 (対応履歴情報を検索)
8. 수주 정보 업데이트 (受注情報を更新)
9. 대응 이력 정보 가져오기 (対応履歴情報を取得)
10. 대응 이력 등록 (対応履歴を登録)
11. 수주 정보 가져오기 (受注情報を取得)
12. 기업 업데이트 (企業を更新)
13. 매출 정보 검색 (売上情報を検索)
14. 매출 정보 등록 (売上情報를등록)
15. 수주 정보 검색 (受注情報を検索)
16. 매출 정보 업데이트 (売上情報を更新)
17. 사안 등록 (案件を登録)
18. 담당자 목록 가져오기 (担当者一覧を取得)
19. 대응 이력 목록 가져오기 (対応履歴一覧を取得)
20. 수주 정보 등록 (受注情報를登録)
21. 수주에 상품 연결 (受注に商品を紐づける)
22. 대응 이력 업데이트 (対応履歴を更新)
23. 사안 정보 가져오기 (案件情報を取得)
24. 기업 정보 가져오기 (企業情報取得)
25. 기업 정보 검색 (企業情報を検索)
26. 사안 목록 가져오기 (案件一覧を取得)
27. 기업 목록 가져오기 (企業一覧を取得)
28. 사안 정보 검색 (案件情報を検索)

## 구현된 트리거
1. 대응 이력이 등록되면 (対応履歴が登録されたら)
2. 사안이 업데이트되면 (案件が更新されたら)
3. 사안이 등록되면 (案件が登録されたら)
4. 수주 정보가 등록되면 (受注情報が登録されたら)
5. 기업이 등록되면 (企業が登録されたら)
6. 수주 정보가 업데이트되면 (受注情報が更新されたら)
7. 대응 이력이 업데이트되면 (対応履歴が更新されたら)

## 테스트 가능 여부
✅ 테스트 가능 (API 기반)

## 테스트를 위한 준비물

1. OAuth 자격증명 필요
2. Next Sfa 계정 (개발자 계정 권장)
3. API 엔드포인트 접근 권한
4. 테스트용 샘플 기업, 사안, 수주 데이터

## 사용 예시

```python
from integration import NextSfaClient

# 클라이언트 설정
client = NextSfaClient()

# 기업 등록
company = await client.register_company(
    name="ABC 株式会社",
    address="東京都渋谷区"
)

# 사안 등록
opportunity = await client.register_opportunity(
    company_id=123,
    opportunity_name="新規システム開発"
)

# 수주 정보 등록
order = await client.register_order(
    opportunity_id=456,
    order_amount=5000000
)
```
